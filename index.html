Config in Env Vars

March 1 2017

David Naughton - UMN Libraries Web Development

naughton@umn.edu

How?

For us, for now, in most cases, put config in .env files...


DB_USER="$user"
DB_PASS="$password"
DB_SERVICE_NAME="hoteltst.oit"

...and use already-existing code to parse them.

Why?

  • Re-use existing code. Delete custom code.
  • Emerging standard: Twelve-Factor App: Config
  • Platform-, framework-, language-agnostic.
  • Easy to override config with local env vars.
